# Portfolio repository for SNHU CS-330

Designing software is a fluid process; each project is unique, and requires a fresh perspective to understand the requirements and desired end product. Because of this, I make sure I know the code fundamentals, and when I understand what the software needs to be, I am able to just write the code without the frustration of trying to learn the fundamentals on-the-fly. I cam quickly apply solutions, without trying to learn the code at the same time.

I enjoyed working with OpenGl in this class. It is a subjet I hadn't played with in the past, and I learned a fun tool which I am sure will be applied to my projects in the future. I know this course skimmed the surface of OpenGL, but it was enough to create interest in 3D graphics, and I look forward to delving deeper.

Learning graphics concepts allowed me to create a custom scene which is near a beginner level, but, I feel, is a strong starting point for creating detailed and interactive 3D environments.

My software development experience hasn't been focused on games, but because of the newly learned OpenGL skills, I find myself thinking of ways to integrate graphics into my portfolio through games, or more likely, through prettier graphs / reports and interactive graphics on data heavy projects, which I largely work on.

Enjoy the scene, I enjoyed making it.
